## Git Branching — Why it matters
main branch = production code (always working)
feature branch = experiment safely
PR/merge = review before adding to main

Databricks engineering workflow:
1. Create branch for each feature
2. Develop + test on branch
3. Open Pull Request
4. Code review by team
5. Merge to main → deploy
