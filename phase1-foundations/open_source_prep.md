# MLflow Open Source Contribution Plan
Date: April 9, 2026

## Why MLflow?
- Databricks built MLflow
- 2-3 merged PRs = top 10% applicant
- Shows platform-specific knowledge
- Resume: "MLflow contributor — PR #XXXX"

## Setup Checklist
- [ ] Fork mlflow/mlflow on GitHub
- [ ] Clone fork locally
- [ ] Install dev dependencies
- [ ] Run test suite (verify setup)
- [ ] Read CONTRIBUTING.md

## Target Issue Types (easiest first)
1. Documentation improvements
   → Fix typos, add examples, clarify docs
   → Zero risk of breaking anything!

2. Test coverage
   → Add missing test cases
   → Well-defined scope

3. Small bug fixes
   → Issues labeled "good first issue"
   → Usually 5-20 lines of code

4. Feature enhancements
   → Labeled "help wanted"
   → More complex but more impact

## Current Issues to Explore
